User: How can I get a pixel from an image.
Your response:
===
# Research
The [[def82a]] article states `v3d.image_get_pixel` can retrieve pixel values at
given coordinates. The coordinates must be 3D regardless of how the image is
being used, so for example a value of 0 should be used for the Z coordinate in a
2D image. The [[a925cd]] article introduces an advanced function
`v3d.image_buffer` which can be used to access the image's internal data buffer.
Additionally, the [[72bb1d]] article explains samplers which can interpolate
pixel values at non-integer coordinates.

# Answer
You can use [[v3d.image_get_pixel]] to get a pixel from an image.
```lua
local pixel_value = v3d.image_get_pixel(my_image, x, y, z)
```
You must pass in all of X, Y, and Z and use 0 for Z if your image is 2D.
A related function is [[v3d.image_buffer]], and you can also use samplers if
you'd like to interpolate values at non-integer coordinates.
===

### Example 2 ###

User: What does a sampler do?
Your response:
===
# Research
The [[72bb1d]] article explains samplers which can interpolate pixel values at
non-integer coordinates. Samplers retrieve the value of neighbouring pixels and,
based on the settings the sampler was created with, can linearly interpolate
those values or select the nearest one.

# Answer
A sampler lets you get pixel values from an image at non-integer coordinates
either by sampling neighbouring values and either linearly interpolating them or
selecting the closest.
===
